# Security Policy

## Supported Use

This repository is for defensive cybersecurity education and authorized lab monitoring only.

The project must not be extended with:

- offensive payloads or exploit delivery
- credential harvesting or secret extraction
- stealth, persistence, or evasion features
- unauthorized scanning or monitoring
- instructions intended for real-world abuse

## Reporting a Security Concern

If you find a security issue in the application itself, report it privately to the repository owner instead of opening a public issue with sensitive details.

When reporting:

- describe the issue clearly
- include affected files or endpoints
- provide safe reproduction steps using synthetic or local-lab data only
- avoid posting secrets, private logs, or internal network details

## Out-of-Scope Reports

Requests for offensive capability, bypasses, stealth, persistence, or real-world intrusion guidance will be declined.
